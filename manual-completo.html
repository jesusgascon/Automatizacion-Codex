Automatizacion-Codex

Sistema local para seleccionar sesiones de Codex, documentarlas mediante resumenes tecnicos y reabrirlas para continuar trabajando desde un unico lanzador de escritorio.

Linux Bash Codex CLI SQLite Lanzador .desktop

1. Que resuelve

Recuperacion

Permite localizar sesiones antiguas por fecha, ruta y titulo sin recordar UUIDs.

Documentacion

Genera un resumen reutilizable de cada sesion con objetivo, trabajo, decisiones y riesgos.

Continuidad

Reabre la sesion real para seguir trabajando sin empezar desde cero.

3. Flujo de funcionamiento

  1. El usuario abre el lanzador del Escritorio.
  2. El script detecta automaticamente el binario de Codex, la base SQLite y la carpeta de Escritorio.
  3. Consulta la tabla threads de la base local de Codex.
  4. Muestra las sesiones principales bajo $HOME, ordenadas por fecha reciente.
  5. El usuario elige una sesion.
  6. El usuario elige entre resumir, abrir, resumir y abrir, archivar o consultar el ultimo resumen existente.

4. Acciones disponibles

Opcion Accion Resultado
1 Generar resumen Crea .txt y .log, muestra vista previa y vuelve al listado.
2 Abrir sesion Ejecuta codex resume <id> y continua la sesion.
3 Resumir y abrir Documenta primero y reabre despues la sesion elegida.
4 Archivar o desarchivar Oculta o recupera una sesion sin borrarla.
5 Ver ultimo resumen Muestra el ultimo resumen guardado sin regenerarlo.

5. Deteccion automatica

Binario Codex

  1. Variable CODEX_BIN si se define manualmente.
  2. command -v codex.
  3. Ultimo binario localizado bajo ~/.nvm/versions/node/*/bin/codex.

Base de sesiones

  1. Variable STATE_DB si se define manualmente.
  2. Ultimo fichero state_*.sqlite localizado en ~/.codex.

Carpeta de Escritorio

  1. xdg-user-dir DESKTOP.
  2. ~/Escritorio.
  3. ~/Desktop.

6. Consulta SQLite

select id, cwd, title, first_user_message, created_at, updated_at, tokens_used
from threads
where cwd = "$HOME" or cwd like "$HOME/%"
  and archived = 0
  and source in ('cli', 'vscode')
order by updated_at desc;

Se excluyen ejecuciones internas para que el selector muestre sesiones de uso real. Los titulos vacios, . o exit se presentan como Sesion sin titulo util, sin ocultarlos.

7. Seguridad

  • No borra sesiones.
  • Solo modifica el estado de archivado cuando el usuario lo confirma.
  • Crea un backup local de la base SQLite antes de archivar o desarchivar.
  • No cambia permisos de proyectos.
  • No crea nuevas sesiones persistentes al resumir gracias a --ephemeral.
  • Separa resumen limpio y log tecnico para facilitar auditoria.
  • Rota backups antiguos y conserva por defecto los 10 mas recientes.

8. Replicacion

El proyecto incluye:

  • instalar.sh para desplegar el lanzador en otro equipo.
  • plantillas/resumir-sesion-codex.desktop.template para generar el acceso directo.
  • Documentacion tecnica completa en docs/.
cd Automatizacion-Codex
bash instalar.sh

9. Configuracion y mantenimiento

Variable Funcion
CODEX_BIN Fuerza la ruta al ejecutable de Codex si la autodeteccion no basta.
STATE_DB Fuerza una base concreta si existen varias.
MAX_BACKUPS Ajusta la retencion de backups previos al archivado.

Tras actualizar el proyecto conviene ejecutar validacion de sintaxis, tests y de nuevo bash instalar.sh para regenerar el lanzador si cambia la plantilla o la ruta.
